Operating Systems features

Process Management:

This is where the programs are loaded to be executed by the CPU. Within a multiprogramming system, the operating system must ensure that the tasks to be executed are being sent within the CPU efficiently. First, some part of the program or the tasks to be executed is loaded within the memory; the operating system ensures when and how much resources it allocates to the CPU for efficient execution. While the task is being run on the CPU, it usually runs for a given time that has been provided through the operating system or until other higher priority tasks take place. Other tasks will be on hold while the higher priority tasks finish executing. The operating system will keep track of the different features available within the CPU, including memory, available storage etc. and then sends jobs through according to the availability of the units.

Memory Management


Each task that the CPU must execute will first be loaded within the RAM; within a single-tasking system, the memory that the system wouldn’t use will be allocated to the given task. While in a multi-tasking system, each area will be given a part of the memory. All operating systems generally use blocks of memory. Each task will be given a block of memory until the task has been completed; then, that part of the memory, the league, will be allocated to another task. Modern operating systems use a method where the disk space can be used as virtual memory; this is where the CPU increases its use of memory to run programs.

File and Disk Management


Files are storage units that the CPU uses to allocate storage spaces and is responsible for handling external storage files. The operating system is responsible for keeping track of each file, where it is located within the storage unit, and how to retrieve them. The operating system is also responsible for being able to redirect files with external storage devices.

Liner Vs. Non-Linear Data Structures

A linear data structure is one in which the items are kept in sequential order, and the components are linked to one another. The information is organised in a logical order and may be retrieved or navigated in a single operation. Because the items are stored in memory in sequential order, linear data structures are easier to implement. An array's data items are traversed one by one and can only access one aspect at a time. Array, Queue, Stack, and Linked List are linear data structures. An array is a collection of data items of the same data type. For example, instead of establishing ten integer type variables to record the roll numbers of ten students, we will build an array with size often. As a result, we may conclude that an array saves a lot of memory and shortens the code.
Stack: A linear data structure that follows the LIFO (Last In-First Out) principle, which means that the data that was inserted last gets deleted first. A push operation adds a data element to a stack, while a pop action is used to add a data element to a pile. In contrast, a pop action removes used to add a data element to a pile, while a pop action is to remove a data piece from the list.
The FIFO rule is applied to a queue (First In-First Out) data structure called a queue (First In-First Out). The piece that was inserted initially shall be deleted first under this rule. In the line, there are two terms: front end and back end. Ad en queue refers to the insertion operation conducted at the back end, whereas de queue refers to the deletion operation performed at the front end.
Lists linked together: A set of nodes consisting of connected together: Aare linked together: It is linked together: It is a set of nodes that consists of two parts: a data element and a reference to the next node in the sequence.
What is the definition of a nonlinear data structure?
Another sort of data structure is a nonlinear data structure, which has data components that are not all contiguous. The data items cannot be traversed or accessed in a single run since the organization is nonsequential. In a linear data structure, an element can be connected to just two other elements (the previous and next element); however, in a nonlinear data structure, an element can be related to several elements.
Nonlinear data structures include trees and graphs. Let's take a closer look at both data structures. A tree is a nonlinear data structure made up of several connected nodes. A graph is a nonlinear data structure with a limited number of vertices and edges, with the edges connecting the vertices. The edges represent the link between the vertices, whereas the vertices are utilized to hold the data components. A graph is employed in a variety of real-world situations, including telephone networks, circuit networks, and social networks such as LinkedIn and Facebook. A single person on Facebook is referred to as a node, and the connections between users are referred to as edges.

MySQL & Databases Usage and Benefits

A Database Management System (DBMS) is a software program that captures and analyses data while interacting with the user, other applications, and the database itself. The database allows for the modification, retrieval, and deletion of data of any type, including strings, integers, photos, etc.
There are primarily four types of DBMS, namely Hierarchical, Relational, Network, and Object-Oriented. Hierarchical DBMS as the name implies, this form of DBMS has a link between predecessors and successors. Consequently, its structure resembles a tree, with nodes representing records and branches representing fields.
This form of DBMS employs a structure that enables users to identify and access data connected to another piece of data in the database.
This type of DBMS enables many-to-many relations that allow many member records to be connected. Object-oriented DBMS employs tiny program components known as objects. Each object comprises a piece of data, and instructions for utilising the data should be used, SQL is the core of a relational database used for access and management. SQL allows you to add, alter, or remove rows of data, obtain subsets of information, modify databases, and conduct numerous other operations. The various SQL subsets are as follows: DDL (Data Definition Language) — This language enables you to conduct numerous database operations, such as CREATE, ALTER, and DELETE.
DML (Data Manipulation Language) enables access to and manipulation of data. It facilitates the insertion, modification, deletion, and retrieval of database data. DCL (Data Control Language) enables you to control database access. Example - Permissions to grant or revoke access.
MySQL is an open-source, cross-platform relational database management system. It supports several storage engines with multi-user access and is powered by Oracle. Oracle offers a commercial license version that includes premium support services. MySQL's plugin libraries allow the database to be embedded into any application. Additionally, it offers stored procedures, triggers, functions, and views for application development and many others. To understand Amazon's RDBMS.
MySQL has robust measures to guarantee that only authorised users may access databases. High Availability - MySQL can run capable of running source/replica replication systems and provides cluster servers at high speeds. Scalability and Flexibility — With MySQL, you can run deeply embedded applications and develop massive data warehouses.
